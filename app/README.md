# 👗 Outfit Archive - 코디 검색 앱

**당신이 가진 옷을 검색하는 것을 중심으로 설계된** 개인용 코디 이미지 아카이빙 웹앱입니다.

"올리브색 가디건 입을까?" → 검색 → 과거 코디 모음 조회

서버 없이 브라우저 내 저장소에만 데이터를 저장하므로 완벽한 프라이버시를 보장합니다.

---

## 🎯 핵심 UX

### 1️⃣ 검색 중심 설계
- 앱을 열면 가장 먼저 검색창이 보입니다
- "올리브 가디건" 입력 → 해당 태그가 있는 이미지들을 3x3 그리드로 즉시 표시
- AND 필터 (기본): 모든 검색어를 포함한 이미지만 노출
- OR 필터: 하나 이상의 검색어를 포함한 이미지 노출

### 2️⃣ 벌크 업로드
- **여러 이미지를 한 번에 추가** 가능
- 드래그&드롭: 폴더 전체를 업로드 구역에 드래그
- 클립보드: Ctrl/⌘+V로 여러 스크린샷 한 번에 붙여넣기
- 파일 선택: 클릭해서 여러 파일 선택
- 진행률 표시로 업로드 상태 확인 가능

### 3️⃣ 갤러리 중심
- **3x3 그리드** 레이아웃으로 한눈에 많은 이미지 표시
- 반응형: 모바일에서는 1열, 태블릿 2열, PC 3열로 자동 조정
- 이미지 카드에서 바로 태그와 메모 확인

### 4️⃣ 빠른 태그 필터링
- 검색 결과의 모든 태그가 **태그 칩**으로 표시됨
- 칩 클릭 시 그 태그로 즉시 필터링
- 이전 검색 태그와 함께 조합 가능

## 📋 주요 기능

### 이미지 관리
✓ **벌크 업로드**: 여러 이미지를 한 번에 추가
✓ **다양한 입력 방식**:
  - 폴더 드래그&드롭
  - 파일 선택 (Ctrl+Click로 다중선택)
  - 클립보드 붙여넣기 (Ctrl/⌘+V)
✓ **자동 썸네일**: 원본과 썸네일을 별도로 저장하여 빠른 로딩
✓ **배치 편집**: 업로드된 이미지는 갤러리에서 바로 클릭해서 태그/메모 추가

### 검색 및 필터
✓ **실시간 검색**: 입력하면서 즉시 결과 반영
✓ **자동완성**: 기존 태그 기반 자동완성
✓ **AND/OR 필터**: 상황에 맞게 선택 가능
✓ **정렬**: 최신순 / 오래된순
✓ **태그 칩 필터**: 검색 결과의 태그를 클릭해서 추가 필터링
✓ **검색 결과 통계**: "3개 / 전체 50개" 형식으로 표시

### 태그 관리
✓ **기본 제공 태그**:
  - 스타일: 꾸안꾸, 꾸꾸꾸
  - 아이템: 가디건, 원피스, 치마, 바지, 코트, 자켓, 니트, 스니커즈, 부츠, 로퍼, 샌들, 가방, 머플러
  - 색상: 흰색, 검정, 회색, 베이지, 브라운, 네이비, 블루, 그린, 올리브, 카키, 레드, 핑크

✓ **커스텀 태그**: 새로운 태그는 자동으로 사전에 등록
✓ **자동완성**: 태그 입력 시 기존 태그 추천

### 편집 및 관리
✓ **상세 편집**: 이미지 클릭 → 풀화면 모달에서 편집
✓ **메모 기능**: 이미지별 텍스트 메모 저장 (갤러리 카드에 미리보기)
✓ **태그 추가/삭제**: 언제든 간편하게 수정 가능
✓ **삭제**: 불필요한 이미지 제거

### 데이터 관리
✓ **내보내기**: JSON으로 전체 데이터 백업 (날짜 자동 포함)
✓ **가져오기**: JSON 복원 (병합 또는 덮어쓰기 선택 가능)
✓ **다중 기기 사용**: 내보내기/가져오기로 기기 간 데이터 이동


4. Settings → Pages → Source: main branch
5. 자동 생성된 URL에서 접속 (예: https://username.github.io/outfit-archive/)

2. "Add new project" 선택

4. 자동 배포

## 💾 데이터 저장 및 백업

### 저장 방식
- **IndexedDB**: 브라우저의 대용량 저장소 (우선 사용)
- **Fallback**: localStorage (IndexedDB 미지원 시)
- **영구 저장**: 브라우저 캐시 삭제 전까지 유지
- **로컬만 저장**: 외부 서버로 데이터 전송 없음

### 백업 방법

**내보내기:**
1. "내보내기" 버튼 클릭
2. JSON 파일 자동 다운로드 (`outfit-archive-2024-01-15.json` 형식)
3. 다운로드 폴더에 저장 (클라우드 동기화 권장)

**복원:**
1. "가져오기" 버튼 클릭
2. 이전 JSON 파일 선택
3. 병합 또는 덮어쓰기 선택
   - **병합**: 기존 데이터 유지, 새 데이터만 추가
   - **덮어쓰기**: 모든 기존 데이터 삭제 후 복원

### 데이터 마이그레이션 (기기 간)
1. 원래 기기: "내보내기" → JSON 다운로드
2. 새 기기: 웹앱 열기 → "가져오기" → JSON 선택
3. "병합" 클릭 → 완료

## 🎮 사용 가이드

### 시나리오: "올리브색 가디건 입을까?"

**Step 1: 앱 열기**
```
URL 입력 → 앱 로드 → 검색창 자동 포커싱
```

**Step 2: 검색**
```
검색창에 입력: "올리브 가디건"
(또는 올리브, 가디건 각각 입력)
```

**Step 3: 결과 확인**
```
3x3 그리드에 해당 이미지들 표시
- 이미지 카드에 태그, 메모, 날짜 표시
- "3개 / 전체 50개" 형식 통계
```

**Step 4: 상세 보기**
```
이미지 클릭 → 풀화면 모달
- 원본 사진 확인
- 태그 추가/삭제
- 메모 작성
- 저장 또는 삭제
```

### 검색 팁

**단일 키워드:**
```
"가디건" 입력 → 가디건 태그 있는 모든 이미지
```

**다중 키워드 (AND):**
```
"올리브 가디건" 또는 "올리브, 가디건"
→ 올리브 AND 가디건을 모두 가진 이미지만
```

**다중 키워드 (OR):**
```
1. 검색창에 "올리브 가디건" 입력
2. "OR 필터" 체크박스 활성화
→ 올리브 OR 가디건을 가진 이미지 모두
```

**빠른 필터링:**
```
검색 후 결과의 태그 칩 클릭
→ 그 태그로 추가 필터링
```

### 이미지 추가

**방법 1: 드래그&드롭 (가장 편함)**
```
1. 파일 탐색기에서 이미지 여러 개 선택
2. 업로드 영역에 드래그
3. 진행률 표시 후 자동 추가
```

**방법 2: 클립보드 붙여넣기**
```
1. 스크린샷 여러 개 (또는 한 개)
2. 앱 페이지 어디서나 Ctrl/⌘+V
3. 바로 갤러리에 추가
```

**방법 3: 클릭으로 파일 선택**
```
1. 업로드 영역 클릭
2. 파일 대화상자 열림
3. Ctrl(또는 Cmd)+클릭으로 여러 파일 선택
4. 열기 → 자동 추가
```

### 태그 추가/편집

**편집 모달에서:**
```
1. 이미지 카드 클릭
2. 큰 모달 창에서 우측 "태그 추가" 섹션
3. 입력창에 태그 입력
4. 자동완성 목록에서 선택하거나 Enter
5. 태그가 칩으로 표시됨
6. 저장
```

**빠른 추가:**
```
태그 입력 → 자동완성에서 선택 → Enter
새 태그도 자동으로 사전에 등록
```

## 🛠️ 트러블슈팅

### Q. 이미지가 저장되지 않음
**A.**
1. 브라우저 저장소 허용 확인
   - 설정 → 개인정보 보호 → 사이트 권한 → 쿠키 및 사이트 데이터
2. 저장소 용량 확인
   ```
   개발자 도구 (F12) → 콘솔에서:
   navigator.storage.estimate().then(e => console.log(e))
   ```
3. 브라우저 캐시 삭제 후 재시도
4. 시크릿/개인정보 보호 모드 사용 X

### Q. 검색이 안 됨
**A.**
1. 정확한 태그명 확인 (대소문자 구분 없음)
2. AND/OR 필터 모드 확인
3. 공백이나 쉼표로 올바르게 구분했는지 확인
4. "✕" 버튼으로 검색 초기화 후 재검색

### Q. 이미지가 로드 안 됨
**A.**
1. 이미지 형식 확인 (JPG, PNG, WebP, GIF 지원)
2. 파일 크기 확인 (너무 크면 시간이 걸림)
3. 브라우저 개발자 도구에서 에러 확인
4. 다른 브라우저로 테스트

### Q. 여러 기기에서 사용하려면?
**A.**
```
1. 기기 A: "내보내기" → JSON 파일 다운로드
2. 기기 B: "가져오기" → JSON 선택
3. "병합" 클릭 → 완료
```

**자동 동기화를 원하면:**
- Google Drive에 JSON 파일 저장
- 다른 기기에서 같은 파일 다운로드 후 가져오기

### Q. 데이터 초기화하려면?
**A.**
```
브라우저 설정 → 캐시/쿠키 삭제 → 이 사이트의 모든 데이터 삭제
```

**또는:**
```
개발자 도구 (F12) → Application → IndexedDB → 삭제
```
