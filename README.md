# Outfit Archive - 코디 이미지 아카이빙 웹앱

개인용 코디 이미지를 태그 기반으로 분류하고 검색할 수 있는 정적 웹앱입니다. 
서버 없이 브라우저 내 저장소(IndexedDB)에 모든 데이터를 영구 저장합니다.

## 주요 기능

### 이미지 관리
- **다양한 입력 방식**: 파일 선택, 드래그&드롭, 클립보드 붙여넣기 (Ctrl/⌘+V)
- **자동 썸네일 생성**: 원본과 썸네일을 별도로 저장하여 로딩 속도 최적화
- **태그 기반 분류**: 사용자가 직접 태그 선택/추가

### 검색 및 필터
- **다중 키워드 검색**: 공백 또는 쉼표로 구분된 태그 검색
- **AND/OR 필터**: AND(기본값) - 모든 태그 포함, OR - 하나 이상의 태그 포함
- **빠른 필터링**: 태그 칩 클릭으로 즉시 필터링
- **정렬 옵션**: 최신순 / 오래된순

### 태그 관리
- **기본 제공 태그**:
  - 스타일: 꾸안꾸, 꾸꾸꾸
  - 아이템: 가디건, 원피스, 치마, 바지, 코트, 자켓, 니트, 스니커즈, 부츠, 로퍼, 샌들, 가방, 머플러
  - 색상: 흰색, 검정, 회색, 베이지, 브라운, 네이비, 블루, 그린, 올리브, 카키, 레드, 핑크
- **자동완성**: 기존 태그를 기반으로 자동완성 제공
- **커스텀 태그**: 언제든 새로운 태그 추가 가능

### 편집 및 관리
- **상세 편집**: 이미지 클릭 시 태그, 메모 추가/삭제
- **메모 기능**: 이미지별 텍스트 메모 저장
- **데이터 백업**: JSON으로 전체 데이터 내보내기
- **데이터 복원**: JSON 파일로 데이터 가져오기 (병합/덮어쓰기 옵션)

## 설치 및 실행

### 1. 로컬 실행 (간단함)

파일 3개를 같은 폴더에 저장:
- `index.html`
- `styles.css`
- `app.js`

브라우저에서 `index.html`을 직접 열면 됩니다.
```bash
# 폴더 경로에서 다음 명령어 실행 (선택사항 - 더 나은 호환성)
python3 -m http.server 8000
# 그 후 http://localhost:8000 접속
```

### 2. 정적 호스팅 배포

#### GitHub Pages
1. GitHub 저장소 생성 (public)
2. 3개 파일(index.html, styles.css, app.js)을 저장소에 업로드
3. Settings → Pages → Source: main branch 선택
4. 제공된 URL에서 접속

#### Netlify
1. [netlify.com](https://netlify.com) 접속
2. "Add new site" → "Deploy manually" 선택
3. 3개 파일을 드래그&드롭
4. 자동 생성된 URL에서 접속

#### Vercel
1. [vercel.com](https://vercel.com) 접속
2. "Add New..." → "Project" 선택
3. 저장소 연결 또는 파일 직접 업로드
4. 자동 배포 후 URL에서 접속

## 데이터 저장 및 백업

### 저장 방식
- **IndexedDB (우선)**: 브라우저 내 대용량 저장소, 영구 저장
- **Fallback**: localStorage (사용 불가 시)
- **모든 데이터는 사용자 브라우저에만 저장** (외부 전송 없음)

### 백업 (내보내기)
1. "내보내기" 버튼 클릭
2. JSON 파일 자동 다운로드 (`outfit-archive-YYYY-MM-DD.json`)
3. 파일을 안전한 곳에 저장

### 복원 (가져오기)
1. "가져오기" 버튼 클릭
2. 이전에 내보낸 JSON 파일 선택
3. "병합" 또는 "덮어쓰기" 선택:
   - **병합**: 기존 데이터 유지, 새 데이터 추가
   - **덮어쓰기**: 모든 기존 데이터 삭제, 새 데이터로 대체

## 사용 가이드

### 이미지 추가
1. **방법 1**: 업로드 영역을 클릭하여 파일 선택
2. **방법 2**: 이미지를 업로드 영역에 드래그&드롭
3. **방법 3**: 스크린샷 후 Ctrl/⌘+V로 붙여넣기

### 이미지 검색
1. 상단 검색창에 태그/키워드 입력 (공백 또는 쉼마로 구분)
2. 자동완성 목록에서 태그 선택
3. 또는 아래 "태그 칩"을 클릭하여 즉시 필터링

### 필터 옵션
- **AND 필터 (기본)**: "하얀치마, 가디건" 입력 시 두 태그를 모두 가진 이미지만 표시
- **OR 필터**: 체크박스 활성화 시 하나 이상의 태그를 가진 이미지 표시

### 이미지 편집
1. 이미지 카드 클릭
2. 팝업에서 태그 추가/삭제, 메모 작성
3. "저장" 또는 "삭제" 선택

## 시스템 요구사항

- **브라우저**: Chrome 24+, Firefox 16+, Safari 10+, Edge 12+
- **저장 공간**: 제한 없음 (브라우저 저장소 용량에 따라, 대부분 수GB)
- **네트워크**: 필요 없음 (오프라인 작동 가능)

## 보안 및 프라이버시

✓ 외부 서버로 데이터 전송 없음
✓ 모든 데이터는 로컬 브라우저에만 저장
✓ 개인정보 수집 없음
✓ HTTPS 미지원 URL에서도 안전 (로컬 저장이므로)

## FAQ

### Q. 이미지가 저장되지 않는다
**A.** 
1. 브라우저 저장소 허용 확인 (설정 → 개인정보)
2. 저장소 용량 확인 (위 참조)
3. 브라우저 캐시 삭제 후 재시도
4. 다른 브라우저로 테스트

### Q. 검색이 안 된다
**A.**
1. 태그가 정확히 입력되었는지 확인 (대소문자 구분 없음)
2. AND/OR 필터 모드 확인
3. 검색창 "✕" 버튼으로 초기화 후 재검색

### Q. 다른 기기에서도 사용할 수 있나
**A.** 
현재는 각 브라우저 저장소가 별도이므로, 다른 기기에서 사용하려면:
1. "내보내기"로 JSON 백업
2. 다른 기기의 웹앱 열기
3. "가져오기"로 JSON 로드

클라우드 싱크가 필요하면 별도 서버 구축 필요 (현재 계획 없음)

### Q. 원본 이미지도 저장되나
**A.** 네, 원본과 썸네일 모두 저장됩니다. 필요시 "내보내기"로 전체 데이터 백업 가능

## 성능 팁

1. **대량 이미지 추가**: 한 번에 여러 개 추가 시 시간이 걸릴 수 있습니다
2. **검색 속도**: 태그가 많을수록 자동완성 시간이 증가하므로, 불필요한 태그 정리 권장
3. **저장소 관리**: 정기적으로 백업하고 오래된 이미지 삭제 권장
